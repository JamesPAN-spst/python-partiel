1. Files (Queues) — FIFO

Une file (queue) est une structure FIFO (First In, First Out) : le premier element ajoute est le premier a sortir. Pensez a une file d'attente a un guichet.

⚠ Piege performance : F.pop(0) est O(n) car tous les elements doivent etre decales ! Pour une file performante, utilisez collections.deque (hors programme mais bon a savoir).
from collections import deque
F = deque()
F.append(5)      # enfiler
F.popleft()      # defiler en O(1) !

✅ Avantage : Toutes les operations sur les piles sont en O(1) avec une liste Python. C'est pourquoi les piles sont souvent preferees aux files pour l'implementation.

Piles et recursivite

Le systeme utilise une pile d'appels (call stack) pour gerer la recursivite. Chaque appel de fonction empile ses variables locales, et le return les depile.

# Exemple : pgcd(9, 6)
# Pile: [9,6] -> empile [6,3] -> empile [3,0]
# return 3 -> depile -> return 3 -> depile -> return 3
💡 Hors programme — Stack overflow : La pile d'appels a une taille limitee (Python : ~1000 appels par defaut). Une recursion trop profonde provoque RecursionError. On peut modifier la limite avec sys.setrecursionlimit(), mais c'est rarement une bonne idee.

3. Application classique : verification du parenthesage

On utilise une pile pour verifier qu'une chaine est correctement parenthesee avec ( ) et [ ].

def bien_parenthesee(s):
    P = []
    for c in s:
        if c in '([':
            P.append(c)
        elif c == ')':
            if len(P) == 0 or P.pop() != '(':
                return False
        elif c == ']':
            if len(P) == 0 or P.pop() != '[':
                return False
    return len(P) == 0
🎓 Au-dela du cours — Notation polonaise inverse (RPN) : Les piles servent aussi a evaluer les expressions en notation postfixee (calculatrices HP). Par exemple, 3 4 + 5 * se calcule : empiler 3, empiler 4, depiler et additionner (7), empiler, empiler 5, depiler et multiplier = 35.
