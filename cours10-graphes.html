1. Definition d'un graphe

Un graphe simple non oriente est un couple (S, A) ou S est un ensemble de sommets et A un ensemble d'aretes reliant des paires de sommets distincts.

⚠ Important : Ne pas confondre le graphe avec son dessin ! Deux dessins differents peuvent representer le meme graphe.

Vocabulaire

TermeDefinition
OrdreNombre de sommets
Degre d'un sommetNombre d'aretes incidentes a ce sommet
Adjacent / VoisinDeux sommets relies par une arete
RegulierTous les sommets ont le meme degre
CompletToutes les aretes possibles existent
ConnexeTout sommet est accessible depuis tout autre
CheminSuite de sommets consecutivement adjacents
CycleChemin ferme dont toutes les aretes sont distinctes
AcycliqueNe contient aucun cycle
💡 Theoreme fondamental : La somme des degres de tous les sommets = 2 x nombre d'aretes (chaque arete est comptee deux fois).

2. Implementation Python (dictionnaire)

Un graphe est represente par un dict : les cles sont les sommets, les valeurs sont les ensembles (set) de voisins.

def graphe_vide():
    return dict()

def ajouter_sommet(G, s, A):
    G[s] = A
    for t in A:
        if t in G:
            G[t].add(s)

def voisins(G, s):
    return G[s]

def sommets(G):
    return G.keys()

def supprimer_sommet(G, s):
    for t in G[s]:
        G[t].remove(s)
    del G[s]

def ajouter_arete(G, s, t):
    G[s].add(t)
    G[t].add(s)

def supprimer_arete(G, s, t):
    G[s].remove(t)
    G[t].remove(s)
✅ Complexite : Avec dict + set, toutes les operations de base sont en O(1) en moyenne (grace au hachage).

3. Matrice d'adjacence

Un graphe d'ordre n peut etre represente par une matrice n x n ou A[i][j] = 1 s'il existe une arete entre i et j, 0 sinon.

ProprieteConsequence sur la matrice
Non orienteMatrice symetrique (A[i][j] = A[j][i])
SimpleDiagonale nulle (A[i][i] = 0)
Degre du sommet iSomme de la ligne i
🎓 Theoreme puissant : Le coefficient (i,j) de Ak donne le nombre de chemins de longueur k entre les sommets i et j. C'est un resultat d'algebre lineaire tres utile !

4. Parcours en profondeur (DFS)

On explore recursivement les voisins non visites. Il faut marquer les sommets visites (contrairement aux arbres) pour eviter les boucles infinies.

def explore(G, s, M):
    # Parcours de la composante connexe contenant s
    M.add(s)              # marquer s
    for t in voisins(G, s):
        if t not in M:
            explore(G, t, M)

def nb_composantes_connexes(G):
    n = 0
    M = set()
    for s in sommets(G):
        if s not in M:
            n += 1
            explore(G, s, M)
    return n

def est_connexe(G):
    return nb_composantes_connexes(G) == 1
⚠ Difference arbre vs graphe : Dans un arbre, on n'a pas besoin de marquer les noeuds (pas de cycle). Dans un graphe, il faut toujours marquer pour eviter de revisiter !

5. Parcours en largeur (BFS)

On explore couche par couche en utilisant une file. Utile pour calculer les distances.

def distance(G, x, y):
    # Retourne la distance de x a y dans G
    F = [x]
    M = set()
    d = 0
    while len(F) > 0:
        C = []                  # couche suivante
        while len(F) > 0:
            s = F.pop(0)
            if s == y:
                return d
            if s not in M:
                M.add(s)
                for t in voisins(G, s):
                    if t not in M:
                        C.append(t)
        d += 1
        F = C
    return None                 # pas de chemin

6. Graphe complet : test

def est_complet(G):
    S = set(sommets(G))
    for s in S:
        if {s} | voisins(G, s) != S:
            return False
    return True

7. Graphes et arbres

Un arbre est un graphe connexe acyclique. Reciproquement, tout graphe connexe acyclique dont on pointe un sommet est equivalent a un arbre enracine.

💡 Propriete fondamentale : Un arbre a n sommets a exactement n-1 aretes. C'est le nombre minimum d'aretes pour un graphe connexe.

8. Chemins euleriens

Un chemin eulerien passe par chaque arete exactement une fois (= tracer sans lever le crayon).

✅ Theoreme d'Euler : Un graphe connexe admet un chemin eulerien si et seulement si il a 0 ou 2 sommets de degre impair.
def a_chemin_eulerien(G):
    if not est_connexe(G):
        return False
    nb_impair = 0
    for s in sommets(G):
        if len(voisins(G, s)) % 2 == 1:
            nb_impair += 1
    return nb_impair in {0, 2}

9. Contenu avance (hors programme)

🎓 Types de graphes :
TypeDescription
SimplePas de boucle (arete d'un sommet vers lui-meme)
OrienteLes aretes ont un sens (x -> y)
PondereLes aretes ont un poids (distance, cout...)
EtiqueteLes aretes portent des labels
MultigraphePlusieurs aretes entre deux memes sommets
🎓 Algorithme de Dijkstra (hors programme) : Calcule le plus court chemin dans un graphe pondere. Utilise une file de priorite (tas). Complexite : O((V+E) log V) avec un tas binaire. C'est l'algorithme utilise par les GPS !
🎓 Chemins hamiltoniens : Un chemin hamiltonien passe par chaque sommet exactement une fois (vs eulerien : chaque arete). Determiner si un tel chemin existe est un probleme NP-complet — pas d'algorithme efficace connu ! Exemples : le probleme du voyageur de commerce.
🎓 Coloration de graphes : Attribuer des couleurs aux sommets de sorte que deux sommets adjacents n'aient pas la meme couleur. Le nombre chromatique est le minimum de couleurs necessaires. Application : emplois du temps, allocation de frequences radio, coloration de cartes (theoreme des 4 couleurs).
