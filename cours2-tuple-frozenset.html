1. tuple — Séquence immutable ordonnée

Un tuple est une collection ordonnée, immutable et hétérogène (types différents autorisés).

t = (1, 'hello', 3.14)    # tuple à 3 éléments
t[0]                        # 1 (indexation)
t[1:]                       # ('hello', 3.14) (slicing)
len(t)                      # 3
⚠ PIÈGE MAJEUR — Tuple à 1 élément :
(1) est un int (parenthèses = groupement mathématique)
(1,) est un tuple à 1 élément (la virgule fait le tuple !)
() est un tuple vide

Immutabilité

On ne peut ni modifier, ni ajouter, ni supprimer un élément :

t = (1, 2, 3)
t[0] = 10       # TypeError !
t.append(4)     # AttributeError !
del t[0]        # TypeError !

Opérations autorisées

(1, 2) + (3, 4)     # (1, 2, 3, 4)  concaténation
(2,) * 3             # (2, 2, 2)      répétition
3 in (1, 2, 3)       # True           appartenance

Déballage (unpacking)

a, b, c = (1, 2, 3)        # a=1, b=2, c=3
a, *b, c = (1, 2, 3, 4, 5) # a=1, b=[2,3,4], c=5
                            # *b capture le reste dans une LISTE
ℹ Note : Le déballage étendu *b crée toujours une liste, même en déballant un tuple.

Hachabilité

Un tuple est hachable si et seulement si tous ses éléments sont hachables :

(1, 2, 'a')       # hachable → peut être clé de dict
(1, [2, 3])        # NON hachable → TypeError si utilisé comme clé
                   # car [2,3] est une liste (mutable)

2. zip et enumerate

zip — Assembler des séquences en parallèle

list(zip([1,2,3], 'abc'))
# [(1,'a'), (2,'b'), (3,'c')]

list(zip([1,2,3], 'ab'))
# [(1,'a'), (2,'b')]  → s'arrête au plus court !
⚠ : zip() renvoie un itérateur, pas une liste. len(zip(...)) → TypeError. Il faut len(list(zip(...))).

enumerate — Indices + valeurs

list(enumerate('abc'))
# [(0,'a'), (1,'b'), (2,'c')]   ordre : (indice, valeur)

3. frozenset — Ensemble immutable

Un frozenset est la version immutable de set. Il est hachable, donc utilisable comme clé de dict ou élément de set.

fs = frozenset({1, 2, 3})
fs.add(4)           # AttributeError ! (immutable)
fs.remove(1)        # AttributeError !

Opérateurs ensemblistes

Les opérateurs &, |, - fonctionnent entre frozenset et set :

frozenset({1,2}) & {2,3}   # frozenset({2})  — type de l'opérande gauche
{2,3} & frozenset({1,2})    # {2}             — type de l'opérande gauche

Cas d'usage principal

# Ensemble contenant un ensemble ? Impossible avec set :
{1, {2, 3}}                 # TypeError: unhashable type 'set'

# Solution avec frozenset :
{1, frozenset({2, 3})}      # OK !
