Cours 3 — Ensembles (set)

→ Quiz (15 questions)

1. Propriétés fondamentales

Un set est une collection non ordonnée, mutable, d'éléments uniques et hachables.

A = {1, 2, 3}
B = set('python')     # {'p','y','t','h','o','n'} — 6 éléments
C = set([1, 1, 2])    # {1, 2} — doublons supprimés
⚠ PIÈGE — Ensemble vide : {} crée un dictionnaire vide, PAS un ensemble ! Utiliser set().

Éléments autorisés

Seuls les types hachables (immutables) sont acceptés :

Autorisé ✓Interdit ✗
int, float, str, tuple, frozensetlist, set, dict
{1, [2, 3]}           # TypeError: unhashable type 'list'
{1, frozenset({2,3})} # OK

2. Opérateurs ensemblistes

OpérateurSignificationExemple
A | BUnion{1,2} | {2,3} = {1,2,3}
A & BIntersection{1,2} & {2,3} = {2}
A - BDifférence{1,2,3} - {2} = {1,3}
A <= BInclusion large (⊆){1,2} <= {1,2,3} = True
A < BInclusion stricte (⊂){1,2} < {1,2} = False
x in AAppartenance2 in {1,2,3} = True
⚠ PIÈGE MORTEL — Priorité des opérateurs :
& > | > - et TOUS s'évaluent AVANT in !

2 in {1,3} & {2,4}
= 2 in ({1,3} & {2,4})   (& d'abord !)
= 2 in set()   (intersection vide)
= False

On s'attendait peut-être à (2 in {1,3}) & {2,4} → False & {2,4}, mais ce n'est PAS ce qui se passe !

Déduplication

{1, 1, 1}             # {1}  — un seul exemplaire
{3, 2, 1} - {1, 1, 1} # {3,2,1} - {1} = {2, 3}

4. Exemple complet (Ex 1 du partiel)

A = {1, 'n'}
A.add('t')         # A = {1, 'n', 't'}
C = set('python')  # C = {'p','y','t','h','o','n'}
A | C              # {1,'n','t','p','y','h','o'}
A & C              # {'n','t'}  (1 n'est pas dans C)
A <= C             # False (1 ∉ C)
C - A              # {'p','y','h','o'}
A.remove(1)        # A = {'n','t'}
A < C              # True ({'n','t'} ⊂ C)
