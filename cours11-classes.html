1. Classe, objet, instance

En Python, tous les types sont des classes et toutes les variables sont des objets (instances de classes). La classe est le "moule", l'objet est la "piece fabriquee".

z = 1 + 2j
type(z)        # <class 'complex'>
z.real         # 1.0   (attribut)
z.imag         # 2.0   (attribut)
z.conjugate()  # (1-2j) (methode)
ConceptDefinition
ClasseModele qui definit attributs, methodes et comportements
Objet / InstanceVariable creee a partir d'une classe
AttributDonnee attachee a un objet (z.real)
MethodeFonction attachee a un objet (z.conjugate())

2. Definir une classe

class Vecteur2d():
    def __init__(self, x, y):   # constructeur
        self.x = x              # attribut x
        self.y = y              # attribut y

    def norm(self):             # methode
        return (self.x**2 + self.y**2)**0.5

    def __repr__(self):         # affichage
        return '(' + str(self.x) + ',' + str(self.y) + ')'
⚠ self : Le premier parametre de chaque methode est self — il designe l'objet lui-meme. On ne le passe pas a l'appel : v.norm() appelle norm(self=v).

Utilisation

u = Vecteur2d(1, 2)   # appelle __init__(self, 1, 2)
u                      # appelle __repr__ -> (1,2)
u.norm()               # 2.2360...
u.x                    # 1  (acces direct a l'attribut)

Surcharge de + pour Vecteur2d

class Vecteur2d():
    # ... __init__, __repr__ ...

    def __add__(u, v):    # surcharge de +
        return Vecteur2d(u.x + v.x, u.y + v.y)

v1 = Vecteur2d(1, 2)
v2 = Vecteur2d(-2.5, 2)
v1 + v2                   # (-1.5, 4)
💡 __rmul__ : Quand Python evalue 3 * v, il essaie d'abord int.__mul__(3, v) qui echoue, puis appelle Vecteur2d.__rmul__(v, 3). C'est le "reverse multiply".

4. Attributs internes (encapsulation)

Prefixer un attribut par __ le rend "interne" (cache) — inaccessible directement depuis l'exterieur.

class Pile():
    def __init__(self):
        self.__P = []      # attribut interne !

    def empile(self, x):
        self.__P.append(x)

    def depile(self):
        assert len(self.__P) > 0, "la pile est vide !"
        return self.__P.pop()

p = Pile()
p.empile(1)
p.__P              # AttributeError ! (acces interdit)
✅ Pourquoi encapsuler ? Pour empecher l'utilisateur de corrompre l'etat interne de l'objet. Par exemple, sans __, on pourrait faire p.P = 0 et casser la pile.
🎓 Hors programme — Name mangling : Python ne cache pas reellement __P. Il le renomme en _Pile__P. C'est du "name mangling". On peut donc y acceder via p._Pile__P, mais c'est une mauvaise pratique.

5. Classe Arbre (avec OOP)

class Arbre():
    def __init__(self, etiquette=None, enfants=[]):
        self.etiquette = etiquette
        assert type(enfants) is list
        self.enfants = enfants

    def __repr__(self):
        return '## ' + str(self.etiquette) \
            + ', ' + str(len(self.enfants)) + ' enfant(s) ##'

A1 = Arbre('a')              # feuille
A2 = Arbre('b')              # feuille
A  = Arbre('c', [A1, A2])    # noeud avec 2 enfants
A.enfants[0].etiquette        # 'a'

Hauteur avec la classe

def hauteur(A):
    H = 0
    for E in A.enfants:       # A.enfants au lieu de enfants(A)
        H = max(H, hauteur(E))
    return H + 1

6. Classe Graphe (avec OOP)

class Graphe():
    def __init__(self):
        self.__G = dict()

    def ajoute_sommet(self, s, voisins):
        assert type(voisins) is set
        self.__G[s] = voisins
        for t in voisins:
            if t in self.__G:
                self.__G[t].add(s)

    def sommets(self):
        return set(self.__G.keys())

    def voisins(self, s):
        return self.__G[s]
💡 Notation objet vs fonction : Avec les classes, on ecrit G.voisins(s) au lieu de voisins(G, s). L'objet est implicitement passe via self.

7. Classe Matrice2 (exemple complet)

Le cours presente une classe Matrice2 pour les matrices 2x2 avec surcharge complete des operateurs.

class Matrice2():
    def __init__(self, a11, a12, a21, a22):
        self.coeff = [[a11, a12], [a21, a22]]

    def __getitem__(self, pos):       # A[i,j] avec indices 1-based
        return self.coeff[pos[0]-1][pos[1]-1]

    def __add__(A, B):                # A + B
        return Matrice2(A[1,1]+B[1,1], A[1,2]+B[1,2],
                        A[2,1]+B[2,1], A[2,2]+B[2,2])

    def __mul__(A, B):                # A * B (produit matriciel)
        return Matrice2(
            A[1,1]*B[1,1]+A[1,2]*B[2,1], A[1,1]*B[1,2]+A[1,2]*B[2,2],
            A[2,1]*B[1,1]+A[2,2]*B[2,1], A[2,1]*B[1,2]+A[2,2]*B[2,2])

    def __rmul__(B, A):               # scalaire * matrice
        return Matrice2(A*B[1,1], A*B[1,2], A*B[2,1], A*B[2,2])

    def __pow__(A, n):                # A**n (exponentiation rapide)
        if n == 0: return Matrice2(1,0,0,1)
        if n == 1: return Matrice2(A[1,1],A[1,2],A[2,1],A[2,2])
        if n % 2 == 1: return (A**(n-1)) * A
        B = A**(n//2)
        return B * B

    def det(self):
        return self[1,1]*self[2,2] - self[1,2]*self[2,1]

    def trace(self):
        return self[1,1] + self[2,2]

    def inv(self):
        d = self.det()
        if d != 0:
            return (1/d) * Matrice2(self[2,2], -self[1,2],
                                    -self[2,1], self[1,1])
🎓 Exponentiation rapide : La methode __pow__ utilise la decomposition binaire de n. Calculer An ne fait que O(log n) multiplications au lieu de n. C'est le meme principe que l'exponentiation rapide pour les entiers.
🎓 Hors programme — Heritage : En POO, une classe peut heriter d'une autre : class Matrice3(Matrice2) reutiliserait les methodes de Matrice2. Python supporte meme l'heritage multiple. C'est un concept fondamental en genie logiciel.
