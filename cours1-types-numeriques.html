1. int — Entiers à précision arbitraire

Contrairement à C ou Java, les entiers Python n'ont aucune limite de taille. On peut manipuler des nombres de milliers de chiffres.

Bases de numération

BasePréfixeExempleConversion
Binaire (2)0b0b1010 = 10bin(10) → '0b1010'
Octal (8)0o0o17 = 15oct(15) → '0o17'
Hexadécimal (16)0x0x1f = 31hex(31) → '0x1f'
⚠ Attention : bin(), oct(), hex() renvoient des chaînes (type str), pas des nombres. type(0b1010) reste int — le préfixe n'est qu'une notation.

Conversion inverse : int('ff', 16) → 255, int('111', 2) → 7.

Opérations mixtes str/int

bin(7) + str(0b11)    # '0b111' + '3' = '0b1113'  (concaténation !)
int('11') - 0b11      # 11 - 3 = 8

2. float — IEEE 754 double précision

Un float occupe 64 bits : 1 bit de signe, 11 bits d'exposant, 52 bits de mantisse. Cela donne environ 16 chiffres significatifs.

Arrondis — Les 4 fonctions

FonctionComportementExemple positifExemple négatif
int(x)Troncature vers zéroint(3.7) = 3int(-3.7) = −3
math.floor(x)Vers −∞floor(3.7) = 3floor(-3.7) = −4
math.ceil(x)Vers +∞ceil(3.7) = 4ceil(-3.7) = −3
round(x)Plus proche (banker)round(3.5) = 4round(-3.7) = −4
⚠ PIÈGE CLASSIQUE : Pour les nombres positifs, int() et floor() donnent le même résultat. La différence n'apparaît qu'avec les négatifs :
math.floor(-2.1) - int(-2.1) = −3 − (−2) = −1

Précision et epsilon machine

La mantisse de 52 bits impose des limites :

SeuilValeurSignification
Epsilon machine y2−52 ≈ 2.22 × 10−16Plus petit 2k tel que 1 + y ≠ 1
Seuil entier x253 ≈ 9 × 1015Plus petit 2k tel que x + 1 == x

Relation fondamentale : x × y = 1

⚠ PIÈGE : 1 + 1e-16 - 1 vaut 0.0, pas 1e-16. L'addition arrondit 1 + 1e-16 à 1.0 car 10−16 < ε/2.

Valeurs spéciales

float('inf')        # infini positif
1e308 * 10          # inf (dépassement vers infini)
float(3**1000)      # OverflowError ! (trop grand pour un float)

3. complex — Nombres complexes

Le suffixe j (pas i) désigne l'unité imaginaire. 2j est un littéral complexe, indépendant de toute variable j.

z = 3 + 4j
z.real               # 3.0  (attribut, PAS une fonction)
z.imag               # 4.0
abs(z)               # 5.0  (module |z| = √(9+16))
import cmath
cmath.phase(z)       # argument de z (en radians)
z ** 0.5             # racine carrée complexe
⚠ PIÈGE : real(z) n'existe PAS en Python ! C'est z.real (attribut sans parenthèses). z.real() avec parenthèses donnerait TypeError car un float n'est pas callable.
ℹ Identité d'Euler numérique : cmath.exp(1j * math.pi) renvoie ≈ (-1+1.2e-16j), soit −1 à la précision machine près.

Piège variable j vs littéral j

j = 1               # variable j = 1
(1 + 2j) ** 2       # 2j est un LITTÉRAL (pas 2*j)
                     # = (1+2j)² = 1 + 4j - 4 = -3+4j

4. Racine carrée entière — Pourquoi la dichotomie

La méthode naïve int(n**0.5) est incorrecte pour les grands entiers :

CasProblème
n = 3**100Résultat incorrect (arrondi float)
n = 3**1000OverflowError (pas convertible en float)

Solution : dichotomie entière

def intsqrt(n):
    a, b = 0, n
    while b - a > 1:
        m = (a + b) // 2
        if m * m <= n:
            a = m
        else:
            b = m
    assert a * a <= n < (a + 1) ** 2
    return a
✓ Avantages : Calcul 100% entier (pas de conversion float), correct pour tout n, complexité O(log n).
