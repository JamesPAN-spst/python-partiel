int — Entiers à précision arbitraire

  • Bases : 0b101 (bin), 0o17 (oct), 0x1f (hex)
  • Conversions : bin(), oct(), hex() → renvoient des chaînes
  • int('ff', 16) → 255  |  int('111', 2) → 7
  • Pas de limite de taille : math.factorial(1000) fonctionne

float — IEEE 754 double précision

  • 64 bits : 1 signe + 11 exposant + 52 mantisse → ~16 chiffres significatifs
  • ⚠ PIÈGE : troncature vs plancher
  • int(-3.7) = -3 (vers zéro)  ≠  math.floor(-3.7) = -4 (vers −∞)
  • ⚠ PIÈGE : perte de précision
  • 1 + 1e-16 - 1 == 0.0 (pas 1e-16 !)
  • Epsilon machine : y = 2−52 ≈ 2.22 × 10−16, plus petit 2k tel que 1 + y ≠ 1
  • Seuil entier : x = 253 ≈ 9 × 1015, plus petit 2k tel que x + 1 == x
  • Relation : x × y = 1
  • float('inf'), 1e308 * 10 → inf
  • Arrondi : round() (plus proche, règle du banker), math.ceil() (vers +∞)

complex — Nombres complexes

  • Littéral : suffixe j → 3+4j, 2j
  • Attributs : z.real, z.imag (pas de parenthèses, pas de fonction globale !)
  • ⚠ PIÈGE : real(z) n'existe PAS → z.real
  • abs(z) = module |z|  |  cmath.phase(z) = argument
  • z**0.5 = racine carrée complexe
  • Euler : cmath.exp(1j * math.pi) ≈ −1

Racine carrée entière (dichotomie)

  • int(n**0.5) : FAUX pour grands n (n = 3100 → erreur d'arrondi)
  • Pour n = 31000 : OverflowError (pas convertible en float)
  • Version correcte : dichotomie sur [0, n], complexité O(log n)
