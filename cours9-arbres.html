1. Definition d'un arbre

Definition iterative

Un arbre est une structure de donnees hierarchique constituee de noeuds. A l'exception d'un noeud special appele racine, chaque noeud a un unique parent. Un noeud peut avoir 0 ou plusieurs enfants. Un noeud sans enfant est une feuille.

Definition recursive

Un arbre est soit vide, soit constitue d'un noeud (la racine) dont les enfants sont eux-memes des arbres.

Vocabulaire essentiel

2. Implementation Python

Chaque noeud est une liste : [etiquette, enfant1, enfant2, ...]

def creer_noeud(e, L):   # etiquette x liste d'enfants -> noeud
    return [e] + L

def etiquette(N):         # noeud -> etiquette
    return N[0]

def enfants(N):           # noeud -> liste de noeuds
    return N[1:]

def creer_arbre(N):       # noeud -> arbre (identifie avec sa racine)
    return N

def racine(A):            # arbre -> noeud
    return A

Exemple

# Construction de l'arbre :
#       X
#      /|\
#     A  R  B
#          / \
#         U   M

U = creer_noeud('U', [])
M = creer_noeud('M', [])
B = creer_noeud('B', [U, M])
A = creer_noeud('A', [])
R = creer_noeud('R', [])
X = creer_noeud('X', [A, R, B])
a = creer_arbre(X)

# Resultat : ['X', ['A'], ['R'], ['B', ['U'], ['M']]]

3. Parcours en profondeur (DFS)

Le parcours en profondeur explore recursivement tous les enfants avant de passer au noeud suivant.

def explore(N):
    # Parcours en profondeur du sous-arbre de racine N
    print(etiquette(N))    # traiter le noeud
    for E in enfants(N):
        explore(E)

Fonctions classiques

def taille(N):
    # Nombre de noeuds du sous-arbre de racine N
    T = 1                  # compte le noeud N
    for E in enfants(N):
        T += taille(E)
    return T

def hauteur(N):
    # Hauteur du sous-arbre de racine N
    T = 0
    for E in enfants(N):
        T = max(T, hauteur(E) + 1)
    return T
✅ Astuce : Pour un arbre a n noeuds, le parcours en profondeur a une complexite O(n) — chaque noeud est visite exactement une fois.

4. Parcours en largeur (BFS)

Le parcours en largeur explore tous les noeuds d'une profondeur donnee avant de passer a la profondeur suivante. On utilise une file :

def parcours_largeur(A):
    F = [racine(A)]          # file initialisee avec la racine
    while len(F) > 0:
        N = F.pop(0)         # extraire le premier
        print(etiquette(N))  # traiter
        for E in enfants(N):
            F.append(E)      # ajouter les enfants a la file
💡 BFS vs DFS : Le parcours en largeur utilise une file (FIFO), le parcours en profondeur utilise implicitement une pile (LIFO, via la recursion). Remplacer la file par une pile dans le BFS donne un DFS iteratif !

L'expression (-1-f(a,b)/cos(x^2))*sin(x) se represente par un arbre ou les noeuds internes sont des operateurs et les feuilles des variables/nombres. Cela facilite la simplification, la derivation et l'evaluation.

6. Contenu avance (hors programme)

🎓 Arbres binaires : Un arbre binaire est un arbre ou chaque noeud a au plus 2 enfants (gauche et droite). C'est la structure la plus etudiee en informatique.

Arbres binaires de recherche (BST)

Un BST est un arbre binaire ou, pour chaque noeud N :

- Tous les noeuds du sous-arbre gauche ont une etiquette < etiquette(N)

- Tous les noeuds du sous-arbre droit ont une etiquette > etiquette(N)

# Recherche dans un BST : O(log n) en moyenne
def recherche_bst(N, x):
    if N is None:
        return False
    if x == etiquette(N):
        return True
    elif x < etiquette(N):
        return recherche_bst(gauche(N), x)
    else:
        return recherche_bst(droite(N), x)
🎓 Parcours infixe d'un BST : Le parcours infixe (gauche, racine, droite) d'un BST produit les elements dans l'ordre croissant. C'est la base du tri par arbre.

Trois ordres de parcours d'un arbre binaire

OrdreVisiteUtilisation
Prefixe (pre-order)Racine, Gauche, DroiteCopie d'arbre, notation polonaise
Infixe (in-order)Gauche, Racine, DroiteTri (BST), expressions infixes
Postfixe (post-order)Gauche, Droite, RacineLiberation memoire, RPN

Tas (Heap)

Un tas max est un arbre binaire complet ou chaque noeud a une etiquette >= celles de ses enfants. Le maximum est toujours a la racine. Utilise pour les files de priorite et le tri par tas (heapsort). En Python : heapq (min-heap).
