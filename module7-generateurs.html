Expressions génératrices vs listes

  • [x**2 for x in range(10)] → list (en mémoire)
  • (x**2 for x in range(10)) → generator (paresseux)
  • ⚠ [(g)] = list contenant un generator, PAS une liste de valeurs !
  • Économie mémoire : sum(x**2 for x in range(10**9)) fonctionne, la version liste tue le processus

Fonctions génératrices (yield)

  • yield au lieu de return → crée un générateur
  • Chaque yield produit une valeur et suspend la fonction
  • ⚠ Un générateur s’épuise ! list(g) une 2e fois → []
  • Peut être récursive

itertools

  • permutations(I) : toutes les permutations
  • permutations(I, k) : arrangements à k éléments
  • combinations(I, k) : combinaisons à k éléments
  • product(A, B) : produit cartésien A×B
  • product(A, repeat=n) : An

Types à distinguer (TRÈS IMPORTANT)

  • [expr for ...] → list
  • (expr for ...) → generator
  • [(expr for ...)] → list contenant un generator
  • (sum(...),) → tuple (virgule !)
  • (sum(...)) → nombre (parenthèses = groupement)
